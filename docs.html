Полная документация AuthMaster

AuthMaster — это учебный проект для демонстрации современных методов аутентификации и авторизации с использованием JWT и OAuth 2.0. В этой документации вы найдете подробные инструкции по настройке, использованию и интеграции системы в ваши приложения.

Введение

AuthMaster предоставляет готовые решения для аутентификации пользователей, включая регистрацию, вход, управление токенами и интеграцию с внешними провайдерами. Проект построен на принципах безопасности и масштабируемости.

  • Версия: 1.0.0 (на 10 октября 2025)
  • Требования: Node.js 14+, PostgreSQL или MongoDB
  • Лицензия: MIT

Установка и настройка

Для начала работы склонируйте репозиторий и установите зависимости:

git clone https://github.com/authmaster/project.git
cd project
npm install

Конфигурация

Создайте файл .env и укажите необходимые переменные:

DATABASE_URL=postgres://user:pass@localhost:5432/authmaster
JWT_SECRET=your_secret_key
OAUTH_GOOGLE_CLIENT_ID=your_google_client_id
OAUTH_GOOGLE_CLIENT_SECRET=your_google_client_secret

Запустите сервер:

npm start

Подробнее о настройке читайте в разделе Установка.

Аутентификация с JWT

AuthMaster использует JWT для внутренней аутентификации. После входа пользователь получает access и refresh токены.

Пример запроса на вход

fetch('https://api.authmaster.example.com/v1/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'user@example.com', password: 'securePassword123' })
})
.then(response => response.json())
.then(data => console.log(data.token));

Подробнее о JWT: JWT документация.

Интеграция с OAuth 2.0

Поддержка провайдеров: Google, GitHub, Facebook. Используйте эндпоинты для инициации и callback.

Пример инициации

window.location.href = 'https://api.authmaster.example.com/v1/oauth/google?redirect_uri=https://yourapp.com/callback';

Подробнее об OAuth: OAuth 2.0 документация.

API референс

Полное описание эндпоинтов API доступно в разделе API. Все запросы используют JSON и требуют Authorization заголовка для защищенных роутов.

Примеры использования

Node.js клиент

const axios = require('axios');

async function login(email, password) {
  const response = await axios.post('https://api.authmaster.example.com/v1/auth/login', { email, password });
  return response.data.token;
}

Часто задаваемые вопросы

Как обработать истечение токена?
Используйте refresh token для получения нового access token через /auth/refresh.
Поддерживается ли двухфакторная аутентификация?
В текущей версии нет, но планируется в обновлении 1.1.
Как настроить кастомные роли?
Используйте админ-эндпоинты для изменения ролей пользователей.
